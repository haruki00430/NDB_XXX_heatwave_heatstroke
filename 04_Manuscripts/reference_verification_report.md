### 修正方針

- **自動置換スクリプト**: 統計値を除外しながら、本文中の引用番号を旧→新にマッピング

### 追加されたルール
1. **バックアップ必須**: 破壊的操作前に `.backup` ファイルを作成
2. **コンテキスト確認**: `grep -B1 -A1` で出現箇所を全確認
3. **3種類のコンテキスト識別**:
   - 本文中の引用: `text.XX)` → 置換対象
   - 統計値: `(N = XX)` → 絶対に置換しない
   - 参考文献リスト: `XX) Author...` → 別途処理
4. **段階的置換**: 参考文献リスト → 本文中の引用（別々に処理）
5. **安全な置換方法**: `replace_all=False` でコンテキスト指定
6. **ロールバック手順**: バックアップからの復元方法

## Phase 3: 本文中の引用番号の置換

### 実施スクリプト
- `replace_citation_numbers.py`: 統計値を除外しながら、引用番号を自動置換

### 置換戦略
1. **プレースホルダー方式**: 旧番号 → `##REF{old_num}##` → 新番号（重複を避ける）
2. **統計値の除外パターン**:
   - `(N = XX)`
   - `p = 0.XXX)`
   - `β = XXX)`
   - `R² = 0.XXX)`
   - `4/N = 0.XXX)` (Cook's distance)
   - 年号（2020, 2021等）
   - 診療行為コード（G004等）
3. **正規表現パターン**: `(?<![0-9(])XX\)(?![0-9])` で前後に数字・括弧がないことを確認

### 置換結果
- **置換された番号**: 32個の旧番号を新番号に置換
  - 例: #5→#4 (1箇所), #12→#11 (2箇所), #33→#29 (4箇所)
- **スキップされた統計値**: 11箇所
  - `(N = 47)` at line 41, 71
  - `(N = 45)` at line 113
  - `(N = 46)` at line 117
  - `(N = 23)` at line 41, 121

## 再発防止のための教訓

### 今回の失敗から学んだこと
1. **`replace_all=true` の危険性**: 引用番号パターン `XX)` は、統計値・参考文献リスト・年号にも出現するため、コンテキストを無視した一括置換は破壊的
2. **段階的置換の重要性**: 参考文献リスト → 本文中の引用を別々に処理することで、破壊を防げる
3. **バックアップの必須性**: 破壊的操作前に必ずバックアップを作成する

- `.claude/skills/manuscript-revision/SKILL.md` に詳細な手順を追加
